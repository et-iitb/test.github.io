Content accuracy and clarity Check if the content is accurate, follows correct symbols, standard norms of representation, uses appropriate terminology and is up-to-date with respect to relevant advances in the topic?

Look at: Audio visuals describing the topic, content, assessments, activities and examples.

Check whether:
  • The content uses scientifically accurate and grade-appropriate scientific information, phenomena, representations and pronunciations
  • The practice questions, activities and assessments (wherever present) are clear and correct.
  • The solutions of the practice questions, activities and assessments (wherever present) are clear and correct.
  • Advances in science are mentioned in the topic which are appropriate for the corresponding grade. For eg. Information about recent disease outbreaks like COVID etc.
Language comprehensibility Check if the language is appropriate and easy to understand for the students?

Look at: the voice over and text used in the video content, text content, assessments, activities and examples

Check whether:
  • The vocabulary used is familiar to the students of the particular age group.
  • An Indian accent is used and easy-to-follow for the students of the particular age group. There should not be use of slang or informal language like gonna, wanna, gotcha.
  • The sentences spoken or those appearing on-screen are short, legible and simple to follow.
  • Vernacular (Regional) language can be used to support comprehension.
Inclusivity Check if the product includes representation of the diverse groups that the students are likely to see around them?

Look at: audio-videos describing the topic, content, assessments, activities, examples.

Check whether:
  • The content includes diverse representations at least in terms of gender, ability (including children with special needs), appearance (such as skin and hair color, and body types), and setting (rural or urban environments).
  • There are no stereotypes associated with the identities presented to the students (e.g., individuals of a certain gender, race, or caste are not depicted in stereotypical professions or roles, such as mothers solely as homemakers or main characters always having a white complexion).
Curriculum alignment and Skill coverage Check if the product is aligned to the targeted curriculum and specific learning goals?

Look at: content of videos, assessments, activities, examples.

Check whether:
  • The product comprehensively covers all the topics outlined in the specific curriculum to which it is mapped (e.g., aligning with the NCERT curriculum if the product is mapped to it).
  • The content covers skills, pedagogy and assessment as prescribed by various bodies like NCF, NEP, NCERT
Bilingual use Note: Only evaluate if your content uses vernacular language. Check if the product includes use of English to introduce students to technical terms (like atoms, cell, organs, light rays etc., laws, names of processes like pasteurization, germination, etc.)?

Look at: content of videos, assessments, activities, examples.

Check whether:
  • The voice-over uses English scientific terms within a vernacular sentence while explaining the concept, such as "Tissue kya hota hai? Group of cell ko hum bolte hai tissue, matlab bahut sare cell jab apas mein combine ho jate hai wo ek tissue ka nirmaad karte hai.”
  • Onscreen, the English terms should be displayed, preferably with transliteration (i.e. English terms written in the vernacular script).
  • For assessment or activity questions, English technical terms should not be omitted from questions in the vernacular language. Vernacular questions should include English translations or incorporate English technical terms in brackets.
Interface Design: Enable Intuitive Use Check if the interface of the product follows principles of interaction design?

Look at: Visual elements such as buttons.

Check whether:
  • The student can easily find the visual elements and functions required to do the given task (e.g., pause, play, exit) and the buttons clearly indicate what function they do.
  • The interface contains sufficient and clear signs informing students about how to use elements of the interface to achieve a desired purpose/goal.
  • The interface is consistent i.e. it follows rules allowing similar operations and using similar elements for achieving similar tasks. There are no exceptions to a rule. (e.g., if the action to highlight any graphical object is by clicking the left mouse button and throughout the product it is the same on the interface)
  • The interface exploits intuitive mappings, meaning the controls to something closely resemble what they affect.
Navigation and pace Check if the interface provides students adequate control over navigation and pace within the product?

Look at: content, assessment, examples, activities

Check whether:
  • The interface contains obvious exit options.
  • The interface allows students to explore the content at their desired pace.
  • A single click will take the student to the desired video, assessment or activity
  • The students can easily move to a certain aspect in the content, e.g., position in the video.
Universal Design Check if the product is accessible to different types of students?

Look at: Audio, video content, assessment, examples, activities

Check whether:
The product at least has the following features:
  • Make it easier for students to see and hear content (for e.g., an option to increase font size)
  • Audio-only or video-only media provides a transcript and/ or on screen text/ caption.
  • Text-only media has a voice-over matching with the text.
  • Provide captions for all prerecorded / live audio (with or without video) content.
  • Possibility of recording an answer and hearing assessment questions as voice-overs.
Analytics for Students’ Progress Check if the interface provides an easy-to-interpret and actionable dashboard with student analytics?

Look at: Dashboard.

Check whether:
  • The dashboard should be visible and easy to find.
  • The dashboard clearly conveys the intended information about the students’ performance, progress, etc.
  • The dashboard should be easily ‘learnable’ (‘Learnable’ means how easy it is for a first-time user to do the basic tasks).
  • The dashboard should provide sufficient guidance to identify where the student needs additional support to make progress.
  • The dashboard analytics is customisable to the needs of the teacher.
  • The dashboard design should support processes including data processing, awareness, reflection and impact.
Content type-technology alignment Check if the visualization type chosen is suitable for teaching the corresponding content type?

Look at: Visualizations such as diagrams, videos, animations or simulations.

Check whether:
  • Diagram is present if content type is fact
  • Animation or video is available if content type is process
  • Simulation is provided if content type is concept/principle
Setting up a Game Environment (Note: Only evaluate Game based learning) Check if the game goals, rules, and game status is clearly communicated to students?

Look at game rules and game goals.
Game Rule should tell How to play the game and
Game Goal should tell what will the student achieve after completing the game?

Check whether:
  • There are one / few rules of the game.
  • Rule(s) are simple to understand by the students.
  • The game rules are effectively communicated prior to playing so that the burden is not on the student to understand the rule.
  • Goal of the game is clear and is clearly communicated to the student.
